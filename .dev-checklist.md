### 3. âœ… Code Quality

Run these commands **in order**:

```bash

npm run lint
npm run lint:fix  # Auto-fix if needed

# 2. Type check - ensure no TypeScript errors
npm run type-check

# 3. Unit tests - ensure all tests pass
npm run test

# 4. Format code
npm run format

# Optional: E2E tests (if UI changed)
npm run test:e2e
```


## Common Scenarios

### Added New Feature
- [ ] Update README.md (Features section)
- [ ] Update CHANGELOG.md (Added section)
- [ ] Update TESTING.md if testing steps changed
- [ ] Bump minor version (1.X.0)
- [ ] Run full checklist

### Fixed Bug
- [ ] Update CHANGELOG.md (Fixed section)
- [ ] Bump patch version (1.1.X)
- [ ] Run lint + type-check + tests

### Updated Dependencies
- [ ] Update CHANGELOG.md (Changed/Updated section)
- [ ] Run tests to ensure compatibility
- [ ] Test in dev mode (`npm run dev`)

### Refactored Code
- [ ] Update TECHNICAL.md if patterns changed
- [ ] Ensure tests still pass
- [ ] No version bump needed (internal change)

---

## Pre-PR Checklist

Before creating a Pull Request:

- [ ] All items from Post-Change Checklist completed
- [ ] CHANGELOG.md updated with all changes
- [ ] README.md reflects current functionality
- [ ] No console.log statements (use console.warn/error)
- [ ] All TODO comments addressed or tracked
- [ ] Tests cover new code
- [ ] Documentation is clear and complete
