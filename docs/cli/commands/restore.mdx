# orphelix restore

Restore Orphelix database from a backup file.

## Usage

```bash
orphelix restore [backup-path]
```

## Safety Features

<Warning>
Before restoring, the current database is automatically backed up as a safety measure.
</Warning>

Safety backup naming:
```
orphelix-backup-pre-restore-{timestamp}.db
```

## Confirmation Prompt

You must confirm the restore operation by typing `y` or `yes`.

Cancel by typing `n` or `N`:

```bash
Continue? [y/N]: n
â­ï¸  Restore cancelled
```

## After Restore

**Restart the application** to load restored settings:

```bash
orphelix restart
```

Changes take effect after restart.

## What Gets Restored

All database tables are restored:

- User settings and preferences
- GitHub integration configuration
- Cluster aliases
- Critical issues monitoring config
- Sidebar customization
- Historical metrics (if applicable)

## Troubleshooting

### Backup File Not Found

```bash
Error: Backup file not found: /path/to/backup.db
```

Check the path and ensure file exists:

```bash
ls ~/orphelix-backups/
```

### Corrupted Backup

```bash
Error: Backup file is corrupted or invalid
```

Try an older backup or contact support.

### Permission Denied

```bash
Error: Permission denied
```

Ensure you have read permissions:

```bash
chmod 644 ~/orphelix-backups/orphelix-backup-*.db
```
