# Orphelix CLI

## Quick Start

```bash
# Install globally via npm
npm install -g orphelix

# Start Orphelix (first run will auto-install and configure)
orphelix start

# Open in browser
orphelix open

# View logs
orphelix logs

# Stop the application
orphelix stop
```

## Access URLs

After starting, access Orphelix at:

- **Custom hostname:** `http://orphelix.local:3000`
- **Standard localhost:** `http://localhost:3000`

## Command Categories

### Process Management
Start, stop, restart, and monitor the application

- [start](/cli/commands/start) - Start Orphelix in background
- [stop](/cli/commands/stop) - Stop Orphelix
- [restart](/cli/commands/restart) - Restart Orphelix
- [status](/cli/commands/status) - Check if running
- [logs](/cli/commands/logs) - View real-time logs
- [list](/cli/commands/list) - List all running instances
- [delete](/cli/commands/delete) - Remove from pm2

### Information
Get information about the application

- [version](/cli/commands/version) - Show version and check for updates
- [info](/cli/commands/info) - Show detailed application info
- [doctor](/cli/commands/doctor) - Run health checks

### Backup & Configuration
Manage data and settings

- [backup](/cli/commands/backup) - Create database backup
- [restore](/cli/commands/restore) - Restore from backup
- [export-config](/cli/commands/export-config) - Export configuration as JSON
- [import-config](/cli/commands/import-config) - Import configuration from JSON

### Development
Tools for developers

- [dev](/cli/commands/dev) - Start in development mode
- [debug](/cli/commands/debug) - Start with debugger
- [test](/cli/commands/test) - Run tests
- [clean](/cli/commands/clean) - Clean caches and logs

### Maintenance
System maintenance and updates

- [update](/cli/commands/update) - Update to latest version
- [startup](/cli/commands/startup) - Enable auto-startup on boot
- [unstartup](/cli/commands/unstartup) - Disable auto-startup

## Global Options

These options work with multiple commands:

- `--port, -p` - Specify custom port (default: 3000)
- `--name, -n` - Instance name for multi-instance support

## Environment Variables

- `ORPHELIX_PORT` - Set default port (can be overridden with `--port`)
