## Overview

The Settings page allows you to customize Orphelix behavior, appearance, and integrations. Access settings from the gear icon in the header or navigate to `/settings`.

## Cluster Configuration

### Cluster Aliases

Give clusters friendly, recognizable names:

<img src="/images/cluster-aliases.png" alt="Cluster Aliases" />

<Steps>
  <Step title="Add Alias">
    Click **"Add Alias"** button
  </Step>

  <Step title="Select Context">
    Choose kubectl context from dropdown
  </Step>

  <Step title="Configure">
    - **Friendly Name**: Display name (e.g., "üöÄ Production")
    - **Description**: Optional notes
    - **Color**: Badge color (optional)
  </Step>

  <Step title="Save">
    Alias appears in context selector
  </Step>
</Steps>

**Example aliases:**
```

gke_project_us-central1_cluster ‚Üí üåé GKE Staging
minikube ‚Üí üíª Local Development
```

### Critical Issues Monitoring

Configure which resource types trigger critical alerts:

<img src="/images/critical-issues-settings.png" alt="Critical Issues Settings" />

**Configurable per resource type:**

<AccordionGroup>
  <Accordion icon="layer-group" title="Deployments">
    Alert when:
    - ‚úÖ No replicas available
    - ‚úÖ Image pull failures
    - ‚úÖ CrashLoopBackOff pods
    
    **Default:** Enabled
  </Accordion>

  <Accordion icon="cube" title="Pods">
    Alert when:
    - ‚úÖ OOMKilled
    - ‚úÖ Failed status
    - ‚úÖ Evicted
    - ‚úÖ High restart count (>10)
    
    **Default:** Enabled
  </Accordion>

  <Accordion icon="server" title="Nodes">
    Alert when:
    - ‚úÖ NotReady status
    - ‚úÖ Disk pressure
    - ‚úÖ Memory pressure
    - ‚úÖ PID pressure
    
    **Default:** Enabled
  </Accordion>

  <Accordion icon="arrows-up-down" title="HPA">
    Alert when:
    - ‚úÖ Cannot scale (no metrics)
    - ‚úÖ At max replicas
    - ‚úÖ Scaling failures
    
    **Default:** Disabled (less critical)
  </Accordion>
</AccordionGroup>

**Alert Settings:**

- **Enable/Disable per type**: Toggle monitoring
- **Severity threshold**: Choose alert level (Warning, Error, Critical)
- **Cooldown period**: Minimum time between alerts (default: 5 minutes)

### Auto-refresh

Configure automatic resource refresh interval:

<Tabs>
  <Tab title="Intervals">
    **Available intervals:**
    - Off (manual refresh only)
    - 10 seconds (real-time)
    - 30 seconds (recommended)
    - 1 minute
    - 5 minutes
    
    **Default:** 30 seconds
  </Tab>

  <Tab title="Behavior">
    When enabled:
    - Resources automatically refresh
    - Real-time status updates
    - Event stream updates
    - No page reload required
    
    <Note>
      Auto-refresh pauses when tab is inactive (saves resources)
    </Note>
  </Tab>

  <Tab title="Per-page">
    Some pages have independent refresh:
    - Dashboard: Auto-refresh enabled
    - Logs viewer: Real-time streaming
    - Events: Auto-update
    
    Can be toggled per page
  </Tab>
</Tabs>

## GitHub Integration

Configure GitHub App for YAML editing and PRs:

<img src="/images/github-integration-settings.png" alt="GitHub Integration Settings" />

### GitHub App Setup

<Steps>
  <Step title="Create GitHub App">
    See [GitHub App Setup](/user/github/github-app-setup) for detailed instructions
  </Step>

  <Step title="Configure Environment Variables">
    Add to `.env.local`:
    
    ```bash
    GITHUB_APP_ID=123456
    GITHUB_APP_CLIENT_ID=Iv1.abc123
    GITHUB_APP_CLIENT_SECRET=secret_here
    GITHUB_APP_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----\n..."
    NEXTAUTH_SECRET=random_secret
    NEXTAUTH_URL=http://localhost:3000
    ```
  </Step>

  <Step title="Login in Settings">
    Click **"Login with GitHub App"** button
    
    Redirects to GitHub for authorization
  </Step>

  <Step title="Select Repository">
    After login, select default repository for YAML editing
  </Step>
</Steps>

### Repository Settings

<AccordionGroup>
  <Accordion icon="folder" title="Default Repository">
    **Purpose:** Repository used for YAML editing
    
    **Selection:**
    - Choose from installed repositories
    - Must have Contents & Pull Requests permissions
    - Can be changed anytime
    
    **Example:** `myorg/k8s-manifests`
  </Accordion>

  <Accordion icon="code-branch" title="Default Branch">
    **Purpose:** Branch to create PRs from
    
    **Options:**
    - main (recommended)
    - master
    - develop
    - Custom branch
    
    **Default:** main
  </Accordion>

  <Accordion icon="folder-tree" title="File Path Patterns">
    **Purpose:** Where to find deployment YAML files
    
    **Examples:**
    - `k8s/deployments/*.yaml`
    - `manifests/production/*.yaml`
    - `*/deployment.yaml`
    
    **Default:** `**/*.yaml` (search all)
  </Accordion>
</AccordionGroup>

## AI Features

Configure OpenAI integration for AI-powered features:

<img src="/images/ai-settings.png" alt="AI Settings" />

### OpenAI API Key

<Steps>
  <Step title="Get API Key">
    Create key at [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
  </Step>

  <Step title="Add to Environment">
    Update `.env.local`:
    
    ```bash
    OPENAI_API_KEY=sk-proj-...
    ```
  </Step>

  <Step title="Verify in Settings">
    Settings shows:
    - ‚úÖ API key configured
    - Model: gpt-4o-mini
    - Features enabled
  </Step>
</Steps>

### AI-Powered Features

When OpenAI configured, enables:

<AccordionGroup>
  <Accordion icon="magnifying-glass" title="Smart File Matching">
    **Feature:** AI finds correct YAML file for deployment
    
    **How it works:**
    1. Analyzes deployment spec
    2. Reads repository files
    3. GPT-4o-mini calculates match scores
    4. Suggests best match
    
    **Fallback:** Pattern-based matching if API unavailable
  </Accordion>

  <Accordion icon="stethoscope" title="Issue Explanations">
    **Feature:** AI explains critical cluster issues
    
    **How it works:**
    1. Detects issue (OOMKill, CrashLoop, etc.)
    2. Collects context (logs, events, specs)
    3. GPT-4o-mini analyzes problem
    4. Provides explanation and solutions
    
    **Example:**
    ```
    Issue: Pod repeatedly OOMKilled
    
    Explanation:
    Your pod is exceeding its 512Mi memory limit due to a 
    memory leak in the application. The Java heap size is 
    configured to use 1GB but the pod limit is only 512Mi.
    
    Solutions:
    1. Increase memory limit to 1Gi
    2. Reduce Java heap size to 400m
    3. Investigate memory leak with profiler
    ```
  </Accordion>

  <Accordion icon="robot" title="Troubleshooting Chat">
    **Feature:** Interactive AI assistant for debugging
    
    **How it works:**
    1. User describes problem
    2. AI asks clarifying questions
    3. Analyzes logs, events, manifests
    4. Suggests solutions step-by-step
    
    **Usage:** Click "AI Troubleshoot" button on resources
  </Accordion>

  <Accordion icon="folder-tree" title="Repository Analysis">
    **Feature:** AI understands repo structure
    
    **How it works:**
    1. Scans repository
    2. Identifies structure (Helm, Kustomize, plain YAML)
    3. Maps deployments to files
    4. Suggests improvements
    
    **Example:**
    ```
    Structure detected: Kustomize
    Base: k8s/base/
    Overlays: k8s/overlays/production/
    
    Mapped deployments:
    - api-gateway ‚Üí k8s/base/api-gateway/deployment.yaml
    - user-service ‚Üí k8s/base/user-service/deployment.yaml
    ```
  </Accordion>
</AccordionGroup>

### AI Usage & Costs

<Tabs>
  <Tab title="Token Usage">
    **Estimated costs** (gpt-4o-mini):
    
    - File matching: ~500 tokens = $0.0001
    - Issue explanation: ~1000 tokens = $0.0002
    - Troubleshooting chat: ~2000 tokens/message = $0.0004
    - Repository analysis: ~5000 tokens = $0.001
    
    **Total typical monthly cost:** $1-5
    
    <Note>
      Costs depend on usage frequency and repository size
    </Note>
  </Tab>

  <Tab title="Rate Limits">
    OpenAI rate limits (Tier 1):
    - 500 requests/minute
    - 30,000 tokens/minute
    
    Orphelix automatically:
    - Queues requests
    - Retries on rate limit
    - Shows progress indicators
  </Tab>

  <Tab title="Privacy">
    **Data sent to OpenAI:**
    - Deployment specifications (YAML)
    - Pod logs (last 100 lines)
    - Event messages
    - File contents from repository
    
    **NOT sent:**
    - Secret values (masked)
    - Passwords or tokens
    - Full cluster state
    
    <Warning>
      Review OpenAI's privacy policy if handling sensitive data
    </Warning>
  </Tab>
</Tabs>

## Saving Settings

<Tabs>
  <Tab title="Auto-save">
    Most settings save automatically:
    - Theme changes
    - Auto-refresh interval
    - Sidebar preferences
    
    **No save button needed**
  </Tab>

  <Tab title="Manual Save">
    Some settings require save:
    - Cluster aliases
    - Notification preferences
    - GitHub repository selection
    
    **Click "Save" button** after changes
  </Tab>

  <Tab title="Persistence">
    Settings stored in:
    - **Browser:** LocalStorage (theme, sidebar)
    - **Database:** SQLite (aliases, notifications)
    - **Environment:** .env.local (API keys)
    
    **Sync:** Settings sync across browser tabs
  </Tab>
</Tabs>

## Exporting/Importing Settings

<Steps>
  <Step title="Export Settings">
    Click **"Export Settings"** button
    
    Downloads `orphelix-settings.json`:
    ```json
    {
      "theme": "dark",
      "visualPreset": "liquid-glass",
      "autoRefresh": 30,
      "aliases": [
        {
          "context": "prod",
          "name": "üöÄ Production"
        }
      ],
      "notifications": {
        "enabled": true,
        "types": ["deployments", "pods", "nodes"]
      }
    }
    ```
  </Step>

  <Step title="Import Settings">
    Click **"Import Settings"** button
    
    Upload `orphelix-settings.json` file
    
    **Overwrites** current settings
  </Step>

  <Step title="Share with Team">
    Distribute settings file to team:
    
    ```bash
    # Commit to repo
    git add .orphelix/settings.json
    git commit -m "Add Orphelix team settings"
    
    # Team members import file
    ```
  </Step>
</Steps>

## Resetting Settings

<Warning>
  Resetting settings cannot be undone. Export first if needed.
</Warning>

<Steps>
  <Step title="Open Reset Dialog">
    Click **"Reset All Settings"** button at bottom
  </Step>

  <Step title="Confirm Reset">
    Checkbox: "I understand this will reset all settings"
    
    Click **"Reset"**
  </Step>

  <Step title="Settings Cleared">
    All settings return to defaults:
    - Theme: Dark
    - Visual Preset: Liquid Glass
    - Auto-refresh: 30s
    - Aliases: None
    - Notifications: Disabled
  </Step>
</Steps>
